# AI4S Plan-First Skill

> 触发命令: `/ai4s-plan` | 版本: 0.1.0 | 灵码团队

## 描述

Plan-First 科学任务规划技能。当用户描述科学计算任务时自动触发，遵循"先规划后执行"原则。

## 触发条件

- 用户描述科学计算、数值模拟、数据分析、论文复现等任务时
- 用户说"帮我做实验"、"计算一下"、"模拟"等关键词时
- 用户说"分析这个数据"、"复现论文"时

## 行为流程

```
用户描述任务
    ↓
① classify_task — 任务分类（领域/类型/复杂度）
    ↓
② generate_plan — 生成执行计划
    ↓
③ 展示计划给用户确认
    ↓ (用户确认)
④ 按步骤执行
    ↓
⑤ 记录状态到 .ai4s/plans/<id>/meta.json
```

## 计划模板

生成的计划使用以下 Markdown 格式：

```markdown
# 🧪 科学计算计划

## 基本信息
- **任务**: {task_description}
- **领域**: {domain}
- **复杂度**: {simple|moderate|complex}
- **预计耗时**: {estimate}
- **创建时间**: {timestamp}

## 任务分类
| 维度 | 分类 |
|------|------|
| 科学领域 | {domain} |
| 任务类型 | {type} |
| 计算规模 | {scale} |
| 精度要求 | {precision} |

## 执行步骤

### Step 1: {step_name}
- **目标**: {objective}
- **方法**: {method}
- **工具**: {tools}
- **预估时间**: {time}
- **检查点**: {checkpoint}

### Step 2: ...

## 风险评估
| 风险 | 概率 | 影响 | 缓解措施 |
|------|------|------|----------|
| {risk} | {low|mid|high} | {1-5} | {mitigation} |

## 依赖项
- [ ] {dependency_1}
- [ ] {dependency_2}

## 验证标准
- [ ] {validation_1}
- [ ] {validation_2}
```

## 领域检查清单

### 流体力学 (CFD)
- [ ] 确定流动类型（层流/湍流/过渡）
- [ ] 检查雷诺数范围，选择合适的湍流模型
- [ ] 确认边界条件（入口/出口/壁面/对称）
- [ ] 网格无关性验证计划
- [ ] 时间步长 CFL 条件检查
- [ ] 收敛残差标准定义

### 材料科学 (DFT/MD)
- [ ] 确定计算方法（DFT/MD/ML势）
- [ ] 选择交换关联泛函（PBE/HSE/...)
- [ ] K-point 和截断能收敛测试
- [ ] 赝势/PAW势选择
- [ ] 结构优化收敛标准
- [ ] 能带/态密度/声子谱等后处理计划

### 机器学习/深度学习
- [ ] 数据集描述和规模
- [ ] 特征工程计划
- [ ] 模型架构选择和理由
- [ ] 损失函数和优化器
- [ ] 训练/验证/测试划分
- [ ] 评估指标定义

### 数值优化
- [ ] 目标函数和约束条件
- [ ] 变量范围和初始猜测
- [ ] 优化算法选择（梯度/启发式）
- [ ] 收敛条件和停止准则
- [ ] 多目标处理策略

### 统计分析
- [ ] 数据分布和预处理
- [ ] 假设检验计划
- [ ] 显著性水平设定
- [ ] 置信区间方法
- [ ] 多重比较校正

### 分子动力学
- [ ] 力场选择（经验势/反应势/极化）
- [ ] 系综选择（NVE/NVT/NPT）
- [ ] 积分器和时间步长
- [ ] 周期性边界条件
- [ ] 温度/压力控制算法
- [ ] 径向分布函数等分析计划

## MCP 工具调用

当 MCP Server 可用时，通过以下工具增强规划：

- `classify_task`: 调用 AI4S MCP Server 的任务分类工具
- `generate_plan`: 调用 AI4S MCP Server 的计划生成工具
- `estimate_resources`: 估算计算资源需求

当 MCP Server 不可用时，基于内置知识完成分类和规划。
